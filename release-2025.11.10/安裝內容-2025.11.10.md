## 二、此版本安裝內容清單

### 2.1 Docker 映像（本版使用）

| 服務 | 映像 | 說明 |
|------|------|------|
| HAPI FHIR | `dear7601/hapi-fhir-jpaserver-starter:2025.11.10` | 本版應用程式（Spring Boot 內嵌伺服器） |
| PostgreSQL (HAPI) | `postgres:15` | HAPI 資料庫 |
| PostgreSQL (Keycloak) | `postgres:15-alpine` | Keycloak 資料庫 |
| Keycloak | `quay.io/keycloak/keycloak:24.0.5` | 身份驗證與 JWT 發行 |

> 若使用自建映像，請以 `docker-compose.example.yml` 為範本，將映像名稱改為您的 Docker Hub 帳號與 tag。

```
└── （其他說明文件見 INDEX.md）
```

### 2.3 必要環境變數（.env）

本版 `docker-compose.yml` 會讀取以下變數，請在 `.env` 中設定：

| 變數 | 說明 | 範例 |
|------|------|------|
| `PG_HAPI_DB` | HAPI 用資料庫名 | `hapi` |
| `PG_HAPI_USER` | HAPI 用資料庫帳號 | `admin` |
| `PG_HAPI_PASS` | HAPI 用資料庫密碼 | （自訂） |
| `PG_KC_DB` | Keycloak 用資料庫名 | `keycloak` |
| `PG_KC_USER` | Keycloak 用資料庫帳號 | `keycloak` |
| `PG_KC_PASS` | Keycloak 用資料庫密碼 | （自訂） |
| `KC_EXTERNAL_PORT` | Keycloak 對外埠 | `8084` |
| `REALM` | Keycloak Realm | `fhir-realm` |

## 三、環境需求

- **Docker**：Docker Engine 20.10+（不需 Docker Desktop）
- **Docker Compose**：2.0+（或 `docker compose` 內建指令）
- **作業系統**：支援 Linux 容器的環境即可，例如：
  - Windows Server（Docker Engine 會以 Linux 容器方式執行）
  - Linux
  - 其他支援 Docker 的伺服器
- **埠**：8080（HAPI）、5432 僅供容器內使用；對外需開放 8080 與 Keycloak 埠（如 8084）。
- **磁碟與記憶體**：足夠供 PostgreSQL、Keycloak、HAPI 三個服務運行。

## 四、此版本安裝步驟

### 4.1 取得本版檔案

- 將 `release-2025.11.10` 整個目錄複製到安裝主機（或解壓縮交付包）。

### 4.2 設定環境變數

1. 進入目錄：
   ```powershell
   cd release-2025.11.10
   ```
2. 若無 `.env`，請在目錄下新增 `.env` 並填入必要變數；或自交付包中複製既有 `.env` 後依環境修改。確認上述「必要環境變數」皆已填寫。

### 4.3 設定授權規則（選用）

- 編輯 `config/authz-rules.json`，依需求設定角色與權限。  
- 若暫不修改，可使用預設內容先完成安裝與驗證。

### 4.4 啟動服務（Docker Compose）

在 **PowerShell** 或 **命令提示字元** 中執行：

```powershell
docker compose up -d
```

或使用舊版語法：

```powershell
docker-compose up -d
```

本版 **不需先建置映像**（使用預設映像 `dear7601/hapi-fhir-jpaserver-starter:2025.11.10`）。若為自建映像，請改用 `docker-compose.example.yml` 並修改映像名稱後再執行。

### 4.5 檢查服務狀態

```powershell
docker compose ps
```

確認 `pg-hapi`、`pg-keycloak`、`hapi`、`keycloak` 皆為 `running`（或 `Up`）。

### 4.6 設定 Keycloak（首次安裝必做）

- 依 [KEYCLOAK_SETUP.md](KEYCLOAK_SETUP.md) 建立 Realm、Client、使用者與角色。  
- 本版 HAPI 使用 JWT，需從 Keycloak 取得 Token 才能呼叫 `/fhir/**`（除健康檢查等例外）。

### 4.7 驗證安裝

- **健康檢查**（不需 Token）：
  ```powershell
  curl http://localhost:8080/actuator/health
  ```
- **授權測試**（需先完成 Keycloak 與測試用戶）：
  ```powershell
  .\test-all-users.ps1
  ```

## 五、Windows Server 注意事項

- 使用 **Docker Engine（Windows Server 版）** 即可，不需 Docker Desktop。
- 容器為 **Linux 映像**，會在 Docker 的 Linux 執行環境中運行。
- 掛載目錄 `./config` 會對應到目前工作目錄下的 `config`，路徑由 Docker 處理。
- 若 Keycloak 的 `host.docker.internal` 無法解析，請在 `.env` 或 Compose 中改為實際主機名稱或 IP。
